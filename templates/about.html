Nous avons réalisé ce projet dans le cadre de l'unité DSIA-4203C - Data Engineering. Le dataset d'APB regroupe des données réparties dans différentes colonnes.

Au cours de ce projet, nous avons commencé par la recherche d'un dataset à la fois intéréssant à réaliser, exploitable et compréhensible. Le choix d'un jeu de donnée sur les affectations APB est particulièrement intéréssant dans la mesure où il s'agit d'une plateforme que nous avons nous même expérimentée et qui ne nous était donc pas inconnue.




Le but du projet est tout d'abord de recueillir des données et de les stocker dans une base de données NoSQL type MongoDB. Pour ce faire, il est nécessaire d'installer l'application de la base de donnée, déjà pour accéder à MongoDB, puis ensuite voir la forme du dataset mais aussi pour voir si le stockage a bien été fait. Ci dessous un comparatif entre une base de données SQL et une NoSQL, au niveau du vocabulaire utilisé.
(Source: mongodb.com)




Ensuite, bien qu'ayant eu une introduction au framework Flask, il a fallu réaliser une interface Web liée à la base de données et permettant de l'interroger. L'appréhension était plus compliquée que prévue. Ce projet est ainsi un premier jet de ce que l'on peut réaliser en data engineering. Il représente un potentiel de développement non négligeable. En effet, on peut continuer sur le recueil de donnée par scraping, croiser les différentes données du dataset afin de mieux préciser les demandes ou même comparer des données de deux datasets différents.
